PLUGIN_SQLITEVIRTUALLIBRARIES_SETTINGS_BROWSEMENUS_ABOUT_TEXT
	EN	Use your own SQLite definition to create virtual libraries and choose from a number of basic menus to browse these virtual libraries.</b><br><br><b>Notes:</b><br><br>- <b>The more virtual libraries you add and the more tracks they contain the more time LMS will need to start or to rebuild your virtual libraries.</b><br>- You <b>don't have to</b> use <i>browse menus</i>. You can just create virtual libraries / library views <b>without</b> any menus.<br>- To <b>delete</b> a virtual library and its menus just <i>select <b>remove</b> as SQLite definition</i> and press <i>apply</i></b>. Or you could just <b>disable</b> them and keep them for later use by unchecking the <i>Enabled</i> box.<br>- Valid <b>menu names</b> can't include special characters like <b>^{}$@<>"#%?*:/|\</b><br>- Your SQLite definition should have a virtual library name and a SQLite statement.<br>- Put your SQLite definitions (file extension: <b>.sql.xml</b>) in the folder named <b>SVLS-VirtualLibrary-definitions</b> which is created in the LMS playlist folder by default. You can change its locations by selecting a different <b>parent</b> folder above.<br><br>Here's an example of a SQLite definition for a virtual library that includes all genres with 'Classical' in their genre names:<br><br>-- VirtualLibraryName: All Classical Genres<br>insert or ignore into library_track (library, track)<br>select '%s', tracks.id from tracks<br>&nbsp;&nbsp;left join genre_track on genre_track.track = tracks.id<br>&nbsp;&nbsp;join genres on genres.id = genre_track.genre<br>&nbsp;&nbsp;where<br>&nbsp;&nbsp;&nbsp;&nbsp;genres.name like '%%Classical%%'<br>&nbsp;&nbsp;&nbsp;&nbsp;and tracks.audio = 1<br>&nbsp;&nbsp;group by tracks.id<br><br><br>
